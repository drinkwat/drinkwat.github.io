什么是Nomachine

​ NoMachine是一款远程桌面软件。适用于Linux、windows、ARM、Android等几乎全系统。它使用户能够方便地远程访问和管理他们的电脑。常见的远程桌面软件还有向日葵、ToDesk等。

​ 官网下载地址

为什么要使用Nomachin

  • 操作简单：NoMachine提供了直观的用户界面，使配置和连接过程变得简单，用户无需有复杂的网络知识就可以轻松进行远程访问。
  • 性能好：NoMachine采用NX技术，相比传统的VNC或RDP等远程桌面技术有更快的速度和更好的性能，特别是在低带宽或高延迟的网络环境下。
  • 安全性：NoMachine提供了多种加密选项，确保远程连接的安全性。它可以配置使用SSL加密进行所有的远程会话，以防止数据泄露。

特别的，当我们操作板卡（如树莓派，json），但是没有显示器时，使用Nomachine进行远程访问是一个不错的选择

原理解释 (不感兴趣直接跳过)

unmae -m

本命令用于查询CPU的系统架构，一般有x86,x86_64,amd64,arm64,arrch64

架构 指令集位数 原因
X86 32位 英特尔出的处理习惯以86为结尾，如80186、80286、80386，所以之后被称之为 X86
AMD64 64位 32位向64位演进时，AMD 抢先制造出了兼容 X86 32位指令集的 64位处理器
X86-64 64位 英特尔设计 IA-64，比amd晚了一步，且不兼容 X86 32位指令集，比较惨淡
因此后续因特尔也采用 AMD64，但在此基础上进行了扩充，并改名为 X86-64
所以 x86_64,x64,AMD64基本上是同一个东西
现在用的intel/amd的桌面级CPU基本上都是x86_64
ARM 一般用于嵌入式系统
ARM-V8 AArch64：64位执行状态
AArch32：32位执行状态		 有两种执行态，适用于 32位和64位
AArch64 64位 从 ARM-V8 中独立出来

可以简单的将 x86_64,x64,AMD64理解为同一种64位的CPU架构，arrch一般用于嵌入式系统。

sudo apt-get update

从服务器获取软件列表，并在本地保存为文件。

在 linux中，安装软件最常用到的是 sudo apt-get ，这个命令会从软件仓库中检索并拉取我们想要的软件。每个linux的发行版，都会维护自己的软件仓库。常用的软件都在仓库中，有固定的维护，保证软件的发行源绝对的安全。

使用这个命令会读取软件仓库的所有源列表，然后保存在本机。方便本机用户检索、对比。相当于刷新本地的软件列表。用户安装软件时，会直接从本地的源列表中，检索出 URL，进行安装。

wget https://download.nomachine.com/download/8.11/Linux/nomachine_8.11.3_4_amd64.deb

在Linux中， wget是一个下载文件的工具，它在终端中使用。对于Linux用户是必不可少的工具

  • 语法

从机的CPU架构为x86_64，且因为ubuntu默认支持deb包，所以使用以上命令安装。

如果安装x86_64版本，则下载的是rpm版本的安装包，在ubuntu系统需要安装rpm包，并将rpm版本的安装包转化为amd64.deb版本，最后安装,较为麻烦，所以没有采用。

sudo dpkg -i nomachine_8.11.3_4_amd64.deb

deb文件是linux发行版debian系统的安装包格式，还有像基于debian系统的发型版ubuntu等系统就是使用的deb格式安装包，我们可以使用dpkg命令进行安装管理这些deb安装包文件。
