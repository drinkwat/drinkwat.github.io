模型整体结构概述

该模型是一个 时空注意力编码器-解码器架构，专为粮仓多传感器温度预测设计，核心思想是 融合空间特征（传感器布局）与时间依赖性（历史序列）。整体流程如下：

输入数据 → Encoder（空间特征提取 + 时序编码） → Decoder（多头注意力 + LSTM） → 温度预测

我们输入的是粮仓历史传感器数据（如过去 72 小时各位置的温湿度），输出 未来温度预测（如未来 24 小时温度）。

我们首先观察重点区域（空间注意力），也就是先看哪些位置的传感器更重要（Cbam），接着我们总结历史规律，分析过去几天温度如何变化（LSTM 和全连接），然后我们发现了某些时刻的数据比较重要，比如某个时间节点突然降温（MultiHeadAttention），最后根据已经总结的规律，一步步预测未来温度（Decoder）

模块逐层解析

通道/空间注意力模块（CBAM）

代码类：ChannelAttention, SpatialAttention, Cbam

  • ChannelAttention：计算 传感器通道重要性（如某些位置的温湿度对预测更关键）
  • SpatialAttention：捕捉 空间位置相关性（如粮仓中心与边缘区域的温度传导关系）
  • Cbam：将通道和空间注意力串联，增强关键时空特征的权重

输入输出：

  • 输入：[batch, time_steps, sensors, features]（如 batch = 32, time = 72, sensors = 10, features = 2）

  • 输出：经过注意力加权的同维度张量

空间特征提取模块（SpatialFeature）

代码类：SpatialFeature

通过 CBAM 提取空间注意力特征，将多维特征展平为时间步序列，供后续时序模型处理

输入输出：

  • 输入：[batch, time_steps, sensors, features]
  • 输出：[batch, time_steps, sensors*features]（空间特征向量化）

编码器（Encoder）

代码类：Encoder

双路时序特征提取：

  • FC 路径：全连接层直接映射特征（快速捕捉局部模式）
  • LSTM 路径：捕捉长期时间依赖（如季节周期性）

输入输出：

  • 输入：[batch, time_steps, sensors*features]
  • 输出：[batch, time_steps, 2*hidden_dim]（双路特征拼接）

多头自注意力（MultiHeadAttention）

  • 代码类：MultiHeadAttention

  • 作用：

    • 在解码阶段 动态对齐不同时间步的重要性（如突发热点事件）
    • 通过多头机制并行学习多种时间依赖模式

  • 输入输出：

    • 输入：queries（当前状态）, keys/values（编码器输出）
    • 输出：[batch, d_model_v]（注意力加权后的上下文向量）

  • 关键创新点：

    • 使用 tanh 替代传统 Softmax，增强 局部时间窗口的聚焦能力
    • 通过 fc_att 层压缩注意力维度，降低计算量

解码器（Decoder）

  • 代码类：Decoder

  • 作用：

    • 自回归预测：结合编码器特征与历史标签，逐步预测未来温度
    • 注意力-LSTM 协同：
      • 用多头注意力筛选关键时间信息
      • LSTM 根据注意力结果和已预测值迭代生成下一步

  • 输入输出：

    • 输入：编码器输出 + 历史标签（label[:, :-1]）
    • 输出：[batch, predict_steps]（未来温度序列）

模型特点和可能的改进点

  1. 时空联合建模：CBAM 处理空间相关性，LSTM+注意力处理时间依赖
  2. 轻量级设计：通道注意力使用 1x1 卷积而非全连接，降低参数量

潜在改进建议

  1. 替换 CBAM 为更高效的注意力模块
    • 使用 SimAM（无参注意力）*或* Coordinate Attention（开源实现：GitHub 链接）
  2. 升级时序模块
    • 用 Informer 或 Autoformer 替换 LSTM，解决长序列预测内存爆炸问题
  3. 引入图神经网络
    • 若传感器位置已知，用 PyG 库的 GATConv 显式建模传感器拓扑关系
