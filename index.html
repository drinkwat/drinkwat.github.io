一定要注意身体，身体优先，当身体不舒服的时候，这个时候如何能让身体舒服就是第一要事，其他滚开，包括考研。

比如在昨天，当时我是中午起来感觉不是很舒服，浑身无力加头晕，当时我在床上躺了一会，大概1个小时作用，然后做了不好的事情，身体更加虚弱了，这个时候我就在想到底是去图书馆还是去洗澡，我犹豫了一会，然后选择去图书馆，后面证明这是非常错误的决定。

首先介绍一下后面发生了什么，当时头有点晕，然后去图书馆，两个搭子都不在，也看不下去书，就想着在图书馆找几本课外书看，看了一会，然后开始看小说（起点），从4点多看到6点半，然后回寝室，吃饭，从7点多看到9点多，就是在床上躺着看小说，后面9点半作用去洗了澡，结束。

然后我们来仔细分析这个过程，首先我们要明白一个事情，我们是大学生，不是高中生，当身体不舒服的时候，我们是会抗拒学习的，我们没有那个毅力去强撑着学习，我们这个时候就应该去休息，如果你不正确休息，那么恭喜你，你就会像上面一样，错误的。不但没有玩好，也没有休息好，白白的浪费时间。

所以我们那个时候的最优解就是，中午起床，感觉不舒服，然后继续休息（但是不应该做手艺活），休息差不多了，感觉还是不行，这个时候就应该去看医生，我觉得应该是洗澡，然后看医生，吃药，然后睡觉，休息一下午，这样我们晚上还能学一会。

不要过重一时的得失，不要想想你得到了什么，要想想你因为这个小得而失去了什么，考研也不是明天就考试，这是拉力赛，你急什么，身体最重要。

总结：

  • 身体最重要，当身体不舒服的时候，就应该休息（睡觉），或者洗澡，或者吃药，你这样想，这是我们法定的休息假日，当然休息的时候少刷抖音啊，毁脑子
  • 不要过重一时的得失，不要想想你得到了什么，要想想你因为这个小得而失去了什么

CBAM

发表于 2025-02-19 更新于 2025-03-05

前言

主要介绍 CBAM: 通道注意力和空间注意力的集成者

注意力机制

把注意力聚集在图像重要区域而丢弃掉不相关的方法。

这个过程是由对于特征自适应权重实现的

注意力机制通常被分为基本四大类：

  • 通道注意力 Channel Attention，告诉网络 what to pay attention to
  • 空间注意力机制 Spatial Attention，告诉网络 where to pay attention to
  • 时间注意力机制 Temporal Attention，告诉网络 when to pay attention
  • 分支注意力机制 Branch Attention，告诉网络 which to pay attention to

最后还有两种混合注意力机制：

  • 通道&空间注意力机制
  • 空间&时间注意力机制

CBAM

通道&空间注意力机制

在给定一张特征图，CBAM 模块能够序列化地在 通道 和 空间 两个维度上产生注意力特征图信息，然后两种特征图信息在与之前原输入特征图进行相乘进行自适应特征修正，产生最后的特征图。CBAM 是一种轻量级的模块，可以嵌入到任何主干网络中以提高性能。

设计启发

当前主要研究：

  • 网络的深度
  • 网络的宽度
  • 网络的维度

深度研究：Lenet VGGNet ResNet

宽度研究： GoogLeNet 中的 Inception 模块

维度研究： Xception ResNeXt

注意力机制的主要目的：

聚焦图像的重要特征，抑制不必要的区域响应

（我个人觉得就是滤波）

我们在在对 通道维度 和 空间维度 组合分析研究，提出了 CBAM 模块，并证实了网络的性能的提升来自于精确的注意力机制和对无相关噪声信息的抑制。

总体流程

CBAM（Convolutional Block Attention Module）是一种结合通道注意力和空间注意力的轻量级模块，通过动态调整特征图的通道和空间权重来增强模型的特征表达能力。其核心由两个子模块组成：

  • 通道注意力模块（Channel Attention Module, CAM）
  • 空间注意力模块（Spatial Attention Module, SAM）

CAM

输入：特征图
输出： 通道注意力权重

实现步骤：

  1. 全局池化

  2. 共享 MLP

  3. 权重融合

  4. 全程池化：

​ 对特征图分别进行 全局平均池化（AvgPool）和 全局最大池化（MaxPool）

$$\begin{aligned}&\mathrm{AvgPool}(F)=\frac{1}{H\times W}\sum_{i=1}^H\sum_{j=1}^WF_{:,i,j}\&\mathrm{MaxPool}(F)=\max_{i,j}F_{:,i,j}\end{aligned}$$

  1. 共享 MLP：

​ 将两个池化结果分别输入一个共享的 多层感知机（MLP），包含两个全连接层（或等效的 1x1 卷积）

$$
\begin{align}
\mathrm{MLP}(\mathrm{AvgPool}(F))
&= W_2\Bigl(\mathrm{ReLU}\bigl(W_1(\mathrm{AvgPool}(F))\bigr)\Bigr) \
\mathrm{MLP}(\mathrm{MaxPool}(F))
&= W_2\Bigl(\mathrm{ReLU}\bigl(W_1(\mathrm{MaxPool}(F))\bigr)\Bigr)
\end{align}
$$

  1. 权重融合：

    将两个 MLP 输出逐元素相加，并通过 Sigmoid 激活函数生成通道注意力权重：

$$M_\mathrm{ت}=\sigma(\mathrm{MLP}(\mathrm{AvgPool}(F))+\mathrm{MLP}(\mathrm{MaxPool}(F)))$$

学习不同通道的重要性，增强对任务关键通道的特征响应

SAM

输入：通道注意力调整后的特征图

输出：空间注意力权重

实现步骤：

  1. 通道维度池化：

​ 沿通道维度分别进行 平均池化 和 最大池化，得到两个 1× H × W 的特征图

  1. 特征拼接与卷积：

​ 将两个池化结果拼接为 2×H×W 的特征图，通过一个7x7卷积层（输出通道为1）生成空间注意力权重：

关注特征图中重要的空间区域，与通道注意力形成互补

特征重标定

将通道注意力和空间注意力权重依次应用于原始特征图：

  1. 通道调整：

​ $$F^{\prime}=M_\mathrm{з}\odot F$$

  1. 空间调整：

    $$F^{\prime\prime}=M_s\odot F^{\prime}$$

最终输出：$F^{\prime\prime}\in\mathbb{R}^{C\times H\times W}$，包含通道和空间维度的注意力增强特征。

小结

我们就秉持调包的思想，对于公式本身，我们肯定不能理解嘛，我们也没这么时间去了解，明白我们的目的是什么，在一个月时间内发论文。

知道是什么，怎么用，怎么调包，用了有什么效果，完了。

我们的创新点就是创新型把这几个点融合进去了，懂否。

CBAM 是一个网络结构，确实是轻量级设计，计算成本低，可灵活嵌入ResNet、VGG等主流CNN架构，在分类、检测、分割任务中均能提升性能。

C 和 C++ 的基本区别

C++ 是 C 的扩展，增加了许多新特性。

C 是面向过程的语言，适合底层开发；C++ 是 C 的超集，支持面向对象编程，功能更强大。

面向过程开发

首先，什么是面向过程：

把程序看成一系列步骤，一步步去执行。

就像做菜，按照食谱一步一步来：洗菜 → 切菜 → 炒菜 → 装盘。

它的程序由函数组成的，其中数据（变量）和操作（函数）是分开的，适合简单、线性的任务。

面向对象开发

C++ 的风格是面向对象编程，把程序看作一组对象，对象有自己的属性和行为。就像开餐厅，餐厅里有厨师、服务员、顾客等对象。每个对象都有自己的职责：

  • 厨师：负责做菜。
  • 服务员：负责上菜。
  • 顾客：负责点菜和吃饭。

其实有点像 python 中一个个类

它的程序由类和对象组成，数据（属性）和操作（方法）封装在一起，适合复杂的系统，易于扩展和维护。

C++ 的面向对象特性

C++ 支持面向对象编程，主要有以下特性：

类（Class）

  • 类是对象的模板，定义了对象的属性和行为。
  • 比如，Dog 类可以有属性 name 和方法 bark()。

封装（Encapsulation）

  • 把数据和方法封装在一起，隐藏内部细节。
  • 比如，你不需要知道手机内部如何工作，只需要知道怎么用。

继承（Inheritance）

  • 一个类可以继承另一个类的特性。
  • 比如，Cat 和 Dog 都可以继承 Animal 类的特性。

多态（Polymorphism）

  • 同一个方法在不同对象中有不同的行为。
  • 比如，Animal 类的 speak() 方法，Dog 会“汪汪”，Cat 会“喵喵”。

总结

  • C：面向过程，适合底层开发，代码简单直接。
  • C++：支持面向对象，适合大型项目，代码更易扩展和维护。
  • 面向过程：关注步骤和流程。
  • 面向对象：关注对象和它们之间的关系。

什么是Nomachine

​ NoMachine是一款远程桌面软件。适用于Linux、windows、ARM、Android等几乎全系统。它使用户能够方便地远程访问和管理他们的电脑。常见的远程桌面软件还有向日葵、ToDesk等。

​ 官网下载地址

为什么要使用Nomachin

  • 操作简单：NoMachine提供了直观的用户界面，使配置和连接过程变得简单，用户无需有复杂的网络知识就可以轻松进行远程访问。
  • 性能好：NoMachine采用NX技术，相比传统的VNC或RDP等远程桌面技术有更快的速度和更好的性能，特别是在低带宽或高延迟的网络环境下。
  • 安全性：NoMachine提供了多种加密选项，确保远程连接的安全性。它可以配置使用SSL加密进行所有的远程会话，以防止数据泄露。

特别的，当我们操作板卡（如树莓派，json），但是没有显示器时，使用Nomachine进行远程访问是一个不错的选择

unmae -m

本命令用于查询CPU的系统架构，一般有x86,x86_64,amd64,arm64,arrch64

架构 指令集位数 原因
X86 32位 英特尔出的处理习惯以86为结尾，如80186、80286、80386，所以之后被称之为 X86
AMD64 64位 32位向64位演进时，AMD 抢先制造出了兼容 X86 32位指令集的 64位处理器
X86-64 64位 英特尔设计 IA-64，比amd晚了一步，且不兼容 X86 32位指令集，比较惨淡
因此后续因特尔也采用 AMD64，但在此基础上进行了扩充，并改名为 X86-64
所以 x86_64,x64,AMD64基本上是同一个东西
现在用的intel/amd的桌面级CPU基本上都是x86_64
ARM 一般用于嵌入式系统
ARM-V8 AArch64：64位执行状态
AArch32：32位执行状态		 有两种执行态，适用于 32位和64位
AArch64 64位 从 ARM-V8 中独立出来

可以简单的将 x86_64,x64,AMD64理解为同一种64位的CPU架构，arrch一般用于嵌入式系统。

sudo apt-get update

从服务器获取软件列表，并在本地保存为文件。

在 linux中，安装软件最常用到的是 sudo apt-get ，这个命令会从软件仓库中检索并拉取我们想要的软件。每个linux的发行版，都会维护自己的软件仓库。常用的软件都在仓库中，有固定的维护，保证软件的发行源绝对的安全。

使用这个命令会读取软件仓库的所有源列表，然后保存在本机。方便本机用户检索、对比。相当于刷新本地的软件列表。用户安装软件时，会直接从本地的源列表中，检索出 URL，进行安装。

wget https://download.nomachine.com/download/8.11/Linux/nomachine_8.11.3_4_amd64.deb

在Linux中， wget是一个下载文件的工具，它在终端中使用。对于Linux用户是必不可少的工具
