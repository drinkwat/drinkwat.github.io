考研之如何和室友相处

发表于 2025-03-19

前言

介于这个题材的敏感性，我们就不在寝室写这个博客了，我们在图书馆写。

事情的起因是，作者今天起晚了，但是之前起的都很早，中午的时候，室友沁硕回寝室，问我是不是一直睡到现在，后面我下楼吃饭，遇到另一个室友阿翔，他虽然面露笑意，但我感觉到有一种讥讽，因为我这几个星期一直都是7点半起床，是我们寝室最早的，就今天起的晚。

我们寝室四个人都在准备考研，我想是不是我之前给他们的压力太大了，他们现在是在看我笑话，你平时不是起的很早吗，怎么今天没起来，还翘课了，你平时不是很得意嘛。

首先，我们来认真分析这件事情，我觉得他们的话可能确实存在讥讽，因为我长期7:30起床的行为构成一个参照杆，当他们发现自己无法持续做到时，可能通过嘲讽来缓解焦虑，这是心理学上的酸葡萄机制，比如阿翔和老段，**阿翔是心思敏感的，我觉得他很可能就是嘲讽我，他发现我出现了松懈，就产生了短暂的心理优势感，可能他也是无意识攻击，是处于长期压抑的备考环境下的过度反应。**至于老段，我看不懂他，他今天叫我起床，这确实是出于好心，我想如果是我的话，我是巴不得这个人一觉睡到下午2点，我自己学个爽，当然也有可能这也是一种嘲讽。至于沁硕，我觉得他没想这么多，他今天叫我起床，上课要交作业，然后又给我通信，虽然中午问我是不是睡到现在，可能是嘲讽吧，但是他的通风报信是挽救了我的今天的，我承他的情，我觉得就算是嘲讽我，那我也认了，随他说吧。

总结来看，阿翔是真正嘲讽，老段可能是有些不满，但是还是好心叫我起床，也有可能是通过帮助他人确认自身优越性（「我能控制自己的作息，还能拯救你」）至于沁硕，我觉得他可能就是随口一问

不可否认的是，我们现在处于高压的考研环境中，任何微妙的互动都可能被放大。我一样，他们也是一样，可能他们真的就是随口一句，被我解读成嘲讽。

但是一定要相信自己的直觉，有的时候真不是自己多心了，多留一份心眼，总归是不错的。

很明显，这种情况到后期会愈演愈烈，我们四个人会相互较劲，今天算是一个苗条，我们该怎么去面对这个事情呢。我们该怎么处理这些事情呢，我们在如何这种高压环境下如何处理室友的嘲讽，同时保持复习进度，避免冲突。

首先，毛主席说过，要团结一切可以团结的朋友，我和室友之间并没有任何竞争关系，他们是我的朋友，与人为善，面带春风，这是我们的行事准则，我们讨论一切的前提，都是基于这个准则，我们四个人应该互帮互助，当然，我肯定不能当冤大头，不能出现他们三个人上岸了，结果我失败了，这就是大傻子。

我们该如何正确处理可能的嘲讽，

注意，他们嘲讽我，说明我们正在领跑，真正的loser根本无人关注。为什么阿翔会嘲讽我，因为我连续几个星期早起，他慌了，所以他嘲讽我，他需要靠嘲讽来换回自己的心安。对于我们来说，“这几天实在是太肝了，放松一下”，我觉得这话一说，他瞬间又慌了，而且这样我们也可以出气。

我们要明白，考研本质上是一场「自我超越赛」，真正的对手从来不在寝室里。我们不需要从室友的反应中确认自我价值。记住：被议论说明我们正在领跑，真正的loser根本无人关注。我可以允许别人有各种想法，但这不影响我的复习进度。学习方源的精神，外界的嘲讽与我何干，埋头做自己的事情。

我们也要尽量减少这种嘲讽，少做容易拉仇恨的行为，与人为善

如果下次我们再晚起，他们问我，你该不会一直睡到现在吧，我们应该这样回复：害，实在是太累了，起不来。同时我们应该主动制造脆弱时刻，比如今天一点都学不下去，或者说今天下午做题目错崩了，心态有问题，我们适度暴露弱点降低他人嫉妒阈值。

同时，我们应该尽量建立起这种非竞争性连接，比如，每周可以适当的分享一些资源给室友，把我们的潜在对手转化为「信息同盟」。不过，要分入，沁硕是知恩图报，阿翔是自私的，老段跟我们没什么关系。如果真的是要分享什么资源，我觉得优先给沁硕比较好。

我们不要在寝室学习，不要在寝室炫耀自己的学习成果，不要说自己的进度，一问就是，学的很慢。

最后，我们应该反思，我们是不是真的给他们带来了压力，我觉得是有一点的，首先我们早上起的动静有点大，我们应该在晚上睡前，就把所有东西都收拾好。同时，晚上睡前，如果有洗漱行为，也应该在室友睡前做完，不能像昨晚一样，所有室友都开始上床了，然后我们才开始洗漱，这样实在是太不礼貌了。

先写这么多吧，剩下的后面再写。

发表于 2025-03-15 更新于 2025-03-19

前言

我们是2月15日开始准备考研的，如今已经过去了一个月，我们首先要简单回顾这一个月到底学了多少，指出这一个月存在的问题，并做出修改。

进度

考研高数：

网课 24节

单词：500词

实际要求：网课 60节

单词：60*30= 1800

严重不合格。

我们是否能在四月份初结束高数基础篇？如果不能结束高数基础篇，那么我们的心态会受到怎么样的影响，我们后面如果焦虑了应该怎么办？

我们大约计算了一下，我们还有90节网课要看，还有15天，也就是平均每天看6节才能在四月末结束。

但我们不会这样，我们不会这样做，一天看6节根本就是囫囵吞枣，也没有意义。

我目前是这样想的，四月份开始专业课计划不变，我们先制定一个15天计划，也就是这两周，这两周唯一休息一整天，其他时间不休息。

也就是说，我们要全天学习14天，我们再抛弃两天用于突发情况，也就是12天

12天平均每天看4节网课，晚上对网课内容进行总结，并复习专业课，也就是说，我们大概看48节网课，同时单词这玩意真傻逼，一点也不想背了，平均一天背100个词，上午复习，中间背半个小时，下午起来背半个小时，晚上刚开始背半个小时。

我们要背诵1200个单词

总结：4月初结束高数基础篇破产，我们大概是在4月中旬结束高数基础篇。

最开始的时候起不来，效率很低，后面生病了，在宿舍躺了一个多星期，后面，也就是这一个多星期，坚持早起，没什么好说的。

洗衣服和洗澡

我们某个星期了出现了一个怪事，由于衣服一直拖着没洗，我们也不想去洗澡，导致有一天早上，早起，但是浑身感觉不舒服

洗完澡，一定要当天洗衣服，我们要知道**，洗衣服是一件很简单的事情，把它从寝室拿到洗衣服，也就是4分钟**，很简单的事情，我们不能因为拖延坏了大事

至于洗澡，我们已经深刻的意识到了它的重要性

  • 隔一天洗一次澡
  • 勤洗头

我们不要对洗头这个事情产生抵触，最多花10分钟的时间，就10分钟。

我们白白花了一天的时间，然后也玩不开心，我们要花好多钱，而且感到很累，最关键还要跟她吵架，吵了也被挂

不要跟她出去，减少跟她出去，跟她出去严格控制时间，就这样。

小说和破

这两个差不多，首先我们一旦看小说，无论是任何小说，都是极大的破坏我们的状态，不要看小说，不要看小说。

如果即时结束任务，这时候想学就学，不想学就玩

这样我们肯定能上岸，加油！！！！

AI推荐

发表于 2025-03-14

deepseek

DeepSeek App 是一款由杭州深度求索公司研发的智能助手类应用，基于先进的大语言模型技术（如 DeepSeek-R1 模型），专注于提供专业级的数据处理、内容创作和复杂问题解答服务。

它目前是国内开发出的世界最强大开源 AI 模型

获取方式

deepseek 可以跨平台使用，主要有两个方式：

  • 电脑端（浏览器 ）
  • 手机端

电脑端

主要是在电脑端浏览器使用

首先打开浏览器

接着在网页搜索 deepseek 官网网址,

按下回车键，进行官网页面

接着我们点击开启对话

这个时候网页跳转到用户注册页面

  • 电话号码注册
  • 微信扫码登录

在这里我们演示电话号码注册

首先输入手机电话号码，接着点击发送验证码

接着网页跳转到智能验证阶段，主要是为了检测当前用户是否是机器人

验证成功后，网页会自动发送验证码到手机端，我们查看验证码，并输入到网页

输入验证码后，点击我已阅读并同意用户协议，接着点击登录按钮

此时我们就可以正常使用 deepseek 了

手机端

  • 下载软件
  • 注册账户
  1. 下载软件

我们主要是在官方的应用商店下载该软件

首先进入应用商店

接着在搜索 deepseek，点击安装

安装成功后，我们打开 deepseek

点击同意用户协议

使用验证码登录，方法和网页版一致

首先我们输入电话号码，并点击发送验证码

接着手机自动进入验证阶段，成功完成验证，后台会自动发送验证码

我们输入验证码，即可以正常使用

使用方式

目前 deepseek 在长文本对话上的能力很强，但是在识图和处理文件的方面明显弱于 openai 等模型

主要使用方式：

  • 深度思考
  • 联网搜索
  • 深度思考 + 联网搜索
  • 普通使用

深度思考（复杂推理与创作）

优势：

  • 长文本连贯性：擅长处理多轮对话、长篇幅逻辑推导（如学术分析、故事创作），上下文记忆优于多数模型。
  • 结构化输出：可生成清晰框架的报告、代码或解决方案，逻辑层次分明。
  • 文化适配性：对中文语境理解更深，尤其在文史哲、本土化内容创作上表现突出。

局限：

  • 依赖文本输入：无法直接分析图像、表格或文件内容（需用户文字描述）。
  • 实时性弱：知识库截止训练时间点，无法主动获取最新信息（需结合联网搜索）。

场景举例：论文大纲撰写、代码 Debug、商业策略分析。

联网搜索（实时信息获取）

优势：

  • 弥补知识时效性：通过联网插件获取新闻、股价、学术进展等最新动态。
  • 精准检索：可指定搜索范围（如特定网站、时间段），提高信息可信度。

局限：

  • 依赖外部工具：需通过 API 或插件接入搜索引擎，响应速度受网络影响。
  • 信息过载风险：需用户明确需求，否则可能返回冗余内容。

场景举例：实时事件追踪、竞品数据对比、旅游攻略更新。

深度思考 + 联网搜索（复合型任务）

优势：

  • 动态知识整合：先检索最新数据，再基于结果进行深度分析（如市场趋势预测）。
  • 多步骤任务自动化：自动完成“搜索 → 筛选 → 推理 → 输出”全流程，减少人工干预。

局限：

  • 复杂度高：需精确设计 Prompt 以协调搜索与分析环节，调试成本较高。
  • 算力消耗大：长文本生成+多轮检索可能导致响应延迟。

场景举例：行业研究报告生成、突发事件影响评估。

普通使用（日常对话与简单问答）

优势：

  • 响应速度快：对简单问题（如定义解释、生活建议）回复直接高效。
  • 多语言支持：中英文切换流畅，适合基础跨语言交流。

局限：

  • 功能单一性：无法处理需图像识别、文件解析的任务（如翻译图片文字、总结 PDF）。
  • 答案泛化：对模糊提问可能返回笼统回答，需用户进一步细化需求。

场景举例：单词翻译、日程建议、趣味闲聊。

总结建议

  • 优先选 DeepSeek：长文本创作、中文深度分析、高性价比场景。
  • 搭配其他工具：需识图/文件处理时，可用 GPT-4+Vision 或专用 OCR 工具预处理，再输入 DeepSeek 分析。
  • 组合使用模式：日常问答用普通模式，专业任务用“深度思考+联网”组合，实现效率最大化。

比如在昨天，当时我是中午起来感觉不是很舒服，浑身无力加头晕，当时我在床上躺了一会，大概1个小时作用，然后做了不好的事情，身体更加虚弱了，这个时候我就在想到底是去图书馆还是去洗澡，我犹豫了一会，然后选择去图书馆，后面证明这是非常错误的决定。

首先介绍一下后面发生了什么，当时头有点晕，然后去图书馆，两个搭子都不在，也看不下去书，就想着在图书馆找几本课外书看，看了一会，然后开始看小说（起点），从4点多看到6点半，然后回寝室，吃饭，从7点多看到9点多，就是在床上躺着看小说，后面9点半作用去洗了澡，结束。

然后我们来仔细分析这个过程，首先我们要明白一个事情，我们是大学生，不是高中生，当身体不舒服的时候，我们是会抗拒学习的，我们没有那个毅力去强撑着学习，我们这个时候就应该去休息，如果你不正确休息，那么恭喜你，你就会像上面一样，错误的。不但没有玩好，也没有休息好，白白的浪费时间。

不要过重一时的得失，不要想想你得到了什么，要想想你因为这个小得而失去了什么，考研也不是明天就考试，这是拉力赛，你急什么，身体最重要。

总结：

  • 身体最重要，当身体不舒服的时候，就应该休息（睡觉），或者洗澡，或者吃药，你这样想，这是我们法定的休息假日，当然休息的时候少刷抖音啊，毁脑子
